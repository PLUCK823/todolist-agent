# Agent TodoList 高保真原型设计

> **交互设计基准：** [打开 V6 可交互页面原型](../../../.superpowers/brainstorm/40507-1783945975/content/workspace-full-flow-v6.html)。实现前必须同时阅读本文档并运行或打开该原型；本文档规定产品规则，V6 原型规定已确认的视觉层级、页面构图、动效手感和交互顺序。

## 1. 设计目标

本轮交付一个桌面端、高保真、可交互的 Agent TodoList MVP 原型。产品以传统任务管理为主工作流，以可展开的 Agent 作为随时可用的协作层。

原型需要验证：

- 用户无需使用 Agent，也能完成任务的创建、查看、编辑、筛选、完成和删除。
- Agent 不打断任务浏览，并能以侧栏、快捷输入框和独立工作区三种方式使用。
- 用户能理解 Agent 的多步执行状态，包括意图理解、接口调用、等待响应、数据同步、成功和失败。
- 导航、弹窗、账户、登录和注册形成完整可点击流程。

本轮以桌面端为主要评审尺寸。移动端实现不在本轮范围内，但组件边界不得阻碍后续响应式适配。

## 2. 产品定位与范围

### 2.1 核心定位

默认体验是“任务优先，Agent 随时可唤起”。任务列表拥有最大的可视面积；Agent 展开时与任务并排，收起后完全释放右侧空间。

### 2.2 原型页面

原型包含以下页面：

1. 我的任务
2. 近期安排
3. 智能助手独立工作区
4. 个人资料
5. 登录
6. 注册

### 2.3 扩展范围说明

登录、注册和个人资料在原 PRD 中属于未来能力。本轮将它们纳入页面原型，用于验证完整入口和退出流程；后续实现计划需将其标为“原型交互”，除非后端同步增加认证接口。

## 3. 全局布局

### 3.1 左侧导航

左侧导航默认以约 68px 的图标轨显示，可展开至约 210px。展开后显示图标、页面名称和登录用户信息。

导航项目：

- 我的任务
- 近期安排
- 智能助手
- 设置
- 用户头像与账户入口
- 导航展开/收起按钮

用户当前所在页面需要有明确的选中状态。展开和收起采用平滑宽度、文字透明度和位移动效。

### 3.2 主工作区

主工作区随左右区域的宽度变化自动伸缩。页面切换使用轻量淡入和垂直位移动效，不使用整页刷新感强烈的动画。

### 3.3 Agent 侧栏

Agent 侧栏展开宽度约 340px，使用深色背景与主任务区形成清晰分层。

- 展开时：紫色 `✦` 位于 Agent 面板头部，点击该符号收起侧栏。
- 收起时：侧栏列宽变为零，深色背景完全消失。
- 收起后：紫色 `✦` 迁移至任务页右上角，排列在“新建任务”右侧；点击后重新展开侧栏。
- 不使用额外的箭头展开/收起按钮。

## 4. 页面设计

### 4.1 我的任务

页面信息层级：

1. 日期、欢迎标题和未完成任务提示
2. 快速询问、新建任务和 Agent 展开入口
3. 全部、进行中、已完成的摘要卡片
4. 搜索、状态筛选和优先级筛选
5. 按“即将到期”“稍后处理”等时间语义分组的任务列表

每条任务显示完成状态、标题、时间/分类、优先级和更多操作。点击任务打开详情弹窗；点击更多操作可进入删除确认。

### 4.2 近期安排

近期安排提供接下来七天的日期条和按时间排列的日程时间线。用户选择日期后，时间线更新为对应日期的任务。页面保留添加安排入口，复用任务创建弹窗。

### 4.3 智能助手独立工作区

该页面用于长对话和连续任务规划，包含：

- 最近对话列表
- 当前会话标题和连接状态
- 用户消息、Agent 回复和结构化操作结果
- 大尺寸输入区
- 新建对话入口

进入独立工作区时，右侧 Agent 侧栏自动收起，避免重复出现两个对话区域。

### 4.4 个人资料

个人资料页显示头像、用户名、邮箱、登录状态、时区、任务统计、Agent 使用次数和账户安全状态。

可交互操作：

- 修改显示名称、邮箱和时区
- 更换头像
- 保存资料
- 退出登录

### 4.5 登录与注册

登录和注册使用左右分栏布局：左侧表达产品价值，右侧承载表单。

登录页包含邮箱、密码、登录按钮、第三方登录占位入口和注册链接。注册页包含显示名称、邮箱、密码、注册按钮和返回登录入口。

退出登录时先显示确认弹窗；确认后退出应用工作区并进入登录页。登录成功后返回应用。注册完成后回到登录页，避免在没有真实认证接口时模拟不明确的自动登录行为。

## 5. 弹窗与浮层

### 5.1 任务弹窗

- 新建任务：标题、描述、截止日期、分类和优先级。
- 任务详情：显示完整任务信息，提供编辑与删除入口。
- 编辑任务：复用新建任务表单结构，标题和主按钮文案随模式变化。
- 删除确认：明确说明不可恢复，必须二次确认。

### 5.2 筛选浮层

状态和优先级使用两个独立浮层。每个浮层内部为单选逻辑，选择后关闭浮层并更新按钮状态。打开一个浮层时自动关闭另一个。

### 5.3 快捷 Agent 输入

按 `Command + K` 或 `Alt + K`，或点击“快速询问”，打开居中的 Agent 输入框。

动效包含：

- 背景模糊和半透明遮罩
- 输入框由轻微缩小、上移状态过渡到正常状态
- 打开后自动聚焦
- `Esc` 或点击遮罩关闭

### 5.4 设置与头像

设置弹窗包含语言、主题、Agent 启动状态和快捷键信息。头像弹窗提供预设头像选择与本地图片上传区域；保存后显示成功反馈。

### 5.5 通用弹窗规则

- 弹窗有明确标题、关闭按钮、取消和主操作。
- 危险操作使用单独颜色和确认文案。
- 关闭方式包括关闭按钮、取消、遮罩和 `Esc`；进行不可中断提交时禁用关闭。
- 保存、删除、登录等操作完成后使用右上角 Toast 反馈。

## 6. Agent 多步执行状态

Agent 对任务的修改不是瞬时黑盒操作。侧栏和独立工作区都应支持展示下列步骤：

1. 理解请求：识别意图和参数。
2. 调用接口：显示工具名称或精简后的接口信息，如 `POST /api/todos`。
3. 等待响应：显示持续时间和运行状态。
4. 同步页面：刷新任务数据并生成自然语言回复。
5. 完成：显示结构化结果卡片。

状态视觉：

- 等待：降低不透明度。
- 运行：旋转指示器，并显示累计耗时。
- 完成：绿色对勾。
- 失败：红色错误标识、简明原因和“重试”入口。

若请求涉及删除等危险操作，Agent 在真正调用接口前必须停在确认步骤，等待用户明确确认。

## 7. 数据流

### 7.1 传统任务操作

用户操作组件 → 前端状态进入提交中 → 调用 Todo API → 成功后刷新列表并关闭弹窗 → 显示 Toast。

失败时保留用户输入和当前弹窗，显示字段级或表单级错误，不清空数据。

### 7.2 Agent 操作

用户消息 → Agent 解析意图 → 前端接收步骤事件 → Agent 调用 Todo API → 前端展示等待状态 → 接收结果事件 → 刷新任务数据 → 展示回复和结构化结果卡片。

建议 Agent 流式事件统一包含：

- `step_started`
- `step_completed`
- `step_failed`
- `confirmation_required`
- `action_completed`

## 8. 错误与边界状态

页面必须设计以下状态：

- 首次使用时的空任务列表
- 搜索或筛选无结果
- 列表加载中
- Todo API 请求失败
- Agent 服务离线
- Agent 工具调用超时
- Agent 操作部分成功
- 登录或注册校验失败
- 上传头像格式或大小不符合要求

Agent 超时不能无限旋转。达到超时时间后显示失败步骤、已等待时间和重试入口，同时保留原始用户消息。

## 9. 动效与可用性

- 侧栏宽度变化使用约 400–500ms 的缓动，内容淡出应早于宽度归零。
- 弹窗使用约 250–320ms 的淡入、缩放和位移动效。
- 页面切换使用约 200–300ms 的轻量淡入。
- Toast 显示约 2–3 秒；错误 Toast 允许手动关闭。
- 遵循 `prefers-reduced-motion`，减少或关闭非必要动画。
- 所有图标按钮具有可读名称和键盘焦点状态。
- 弹窗打开后焦点进入弹窗，关闭后返回触发元素。
- 不仅依赖颜色表达优先级、成功或失败状态。

## 10. 组件边界

建议前端拆分为：

- `AppShell`：左右栏和主页面容器
- `NavigationRail`：左侧导航状态
- `AgentPanel`：Agent 侧栏及多步状态
- `CommandPalette`：快捷 Agent 输入
- `TaskDashboard`：任务首页
- `UpcomingPage`：近期安排
- `AssistantPage`：独立 Agent 工作区
- `ProfilePage`：账户资料
- `AuthPage`：登录与注册
- `TaskDialog`、`ConfirmDialog`、`SettingsDialog`、`AvatarDialog`
- `FilterPopover`、`ToastRegion`

各组件通过明确的 props 和事件通信；页面组件不直接实现 API 请求细节。

## 11. 验证标准

高保真原型至少支持以下完整路径：

1. 创建任务 → 保存 → 列表反馈。
2. 打开任务 → 编辑或删除 → 二次确认 → 状态反馈。
3. 状态和优先级筛选。
4. 左侧导航展开/收起和页面切换。
5. Agent 展开/完全收起，以及快捷输入框。
6. Agent 多步执行与等待状态。
7. 更换头像和保存资料。
8. 退出确认 → 登录页 → 注册页 → 返回登录 → 登录回到应用。

原型评审通过的核心标准是：用户能够明确知道自己在哪个页面、操作是否正在执行、Agent 当前进行到哪一步，以及操作成功或失败后下一步能做什么。

### 12.2 目标视口与布局尺寸

视觉基准视口为桌面端 `1223 × 1227px`。页面应在 `1000px` 以上宽度保持三列工作区能力。

| 区域 | 默认尺寸 | 展开/收起规则 |
| --- | --- | --- |
| 页面外边距 | 14px | 小尺寸桌面可压缩至 8px |
| 左侧导航 | 68px | 展开约 210px |
| Agent 侧栏 | 340px | 收起后为 0px，不保留深色窄栏 |
| 应用圆角 | 20px | 小尺寸桌面可使用 15px |
| Agent 展开入口 | 38 × 38px | 收起后位于“新建任务”右侧 |
| 弹窗宽度 | 最大 520px | 快捷 Agent 输入最大 630px |

主工作区使用弹性宽度。左侧导航和 Agent 侧栏变化时，主工作区必须连续伸缩，不能横向溢出或保留空白占位。

### 12.4 动效参数

| 场景 | 时长 | 缓动/表现 |
| --- | --- | --- |
| 左右侧栏宽度变化 | 450–500ms | `cubic-bezier(.22,1,.36,1)` |
| 弹窗打开 | 300ms | 淡入、轻微上移和 `0.96 → 1` 缩放 |
| 页面切换 | 300ms | 淡入并从下方移动约 8px |
| Agent 入口出现 | 380ms | 淡入、横向移动和轻微缩放 |
| Toast 出现 | 300ms | 从上方移动并淡入 |
| Agent 步骤旋转器 | 700ms/圈 | 线性循环 |

所有动效必须提供 `prefers-reduced-motion` 降级：移除大部分位移和缩放，只保留必要的透明度变化或直接切换。

### 12.5 页面与状态映射

实现至少需要以下状态：

| 状态 | 可见区域 | 进入方式 |
| --- | --- | --- |
| `tasks` | 我的任务页 | 默认登录后页面 |
| `upcoming` | 近期安排页 | 左侧近期安排入口 |
| `assistant` | 独立 Agent 工作区 | 左侧智能助手入口 |
| `profile` | 个人资料页 | 点击左侧用户头像 |
| `login` | 登录页 | 退出确认后进入 |

| `agentCollapsed` | Agent 侧栏宽度为零 | 点击展开面板中的紫色 `✦` |
| `commandOpen` | 快捷 Agent 输入框 | `Command/Alt + K` 或快速询问 |

进入 `assistant`、`profile` 或 `upcoming` 页面时可自动收起 Agent 侧栏，避免压缩非任务页面；返回 `tasks` 时保留用户最近一次 Agent 展开状态。

### 12.8 Mock API 与延迟

在没有真实后端时，原型使用内存数据完成交互。建议模拟：

| 操作 | 延迟 |
| --- | --- |
| 普通任务查询 | 300–500ms |
| 新建/编辑任务 | 500–800ms |
| 删除任务 | 400–700ms |
| Agent 意图理解 | 700–900ms |
| Agent 调用 Todo API | 700–900ms |
| Agent 等待接口返回 | 1200–1600ms |

Agent 多步演示完成后必须向任务列表写入结果或明确模拟同步反馈，不能只播放动画。另需提供可触发的失败模式，至少覆盖接口超时和参数校验失败。

### 12.9 实施完成定义

其他 AI 编程工具只有在满足以下条件后才能认为“完成原型”：

- 通过第 11 节的八条完整路径。
- 页面视觉密度、区域比例和关键颜色与 V6 原型一致。
- 所有批注过的控件均可交互，没有无响应按钮。
- Agent 收起后右侧深色栏完全消失。
- 登录、注册、退出、头像、筛选和 Agent 多步状态形成闭环。
- 键盘可访问，弹窗焦点行为正确。
- 在目标桌面视口下无横向溢出、遮挡或布局跳动。
